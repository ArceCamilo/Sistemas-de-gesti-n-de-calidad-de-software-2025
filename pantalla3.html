Ciclo de Vida del Desarrollo de Software (SDLC)

El SDLC es un proceso estructurado que define las fases para desarrollar software de alta calidad

📋

Planificación y Análisis

Definición de requisitos, análisis de factibilidad y planificación del proyecto

🎨

Diseño

Arquitectura del sistema, diseño de interfaces y especificaciones técnicas

💻

Implementación

Codificación del software según las especificaciones de diseño

🧪

Pruebas

Verificación y validación del software para asegurar la calidad

🚀

Despliegue

Instalación y puesta en producción del software

🔧

Mantenimiento

Soporte, corrección de errores y mejoras continuas

Modelos Tradicionales

Conoce los modelos clásicos de desarrollo de software, sus características y cuándo utilizarlos

Modelo en Cascada

✅ Ventajas

  • • Fácil de entender y gestionar
  • • Documentación completa
  • • Ideal para proyectos estables
  • • Fases bien definidas

❌ Desventajas

  • • Inflexible a cambios
  • • Detección tardía de errores
  • • No hay entregas tempranas
  • • Riesgo alto en proyectos largos

🎯 Cuándo Usar

  • • Requisitos bien definidos
  • • Proyectos pequeños
  • • Tecnología conocida
  • • Regulaciones estrictas

Modelo en V

✅ Ventajas

  • • Detección temprana de defectos
  • • Pruebas planificadas desde el inicio
  • • Mejor trazabilidad
  • • Enfoque en verificación

❌ Desventajas

  • • Rígido como cascada
  • • Costoso de cambiar
  • • No hay prototipos tempranos
  • • Requiere requisitos estables

🎯 Cuándo Usar

  • • Sistemas críticos
  • • Requisitos claros y estables
  • • Proyectos con alta calidad
  • • Regulaciones de seguridad

Modelo Iterativo e Incremental

✅ Ventajas

  • • Entregas tempranas y frecuentes
  • • Feedback continuo del usuario
  • • Reducción de riesgos
  • • Flexibilidad para cambios

❌ Desventajas

  • • Requiere más recursos
  • • Gestión más compleja
  • • Puede ser costoso
  • • Necesita equipos experimentados

🎯 Cuándo Usar

  • • Proyectos grandes y complejos
  • • Requisitos cambiantes
  • • Necesidad de entregas rápidas
  • • Equipos experimentados

Metodologías Ágiles

🏉 Scrum

👥 Roles

  • • Product Owner
  • • Scrum Master
  • • Development Team

📋 Artefactos

  • • Product Backlog
  • • Sprint Backlog
  • • Product Increment

🎯 Ceremonias

  • • Sprint Planning
  • • Daily Scrum
  • • Sprint Review
  • • Sprint Retrospective

⚡ Principios

  • • Transparencia
  • • Inspección
  • • Adaptación

📋 Kanban

📊 Principios

  • • Visualizar el trabajo
  • • Limitar WIP
  • • Gestionar el flujo
  • • Hacer políticas explícitas

🎯 Beneficios

  • • Flexibilidad
  • • Flujo continuo
  • • Reducción de desperdicios
  • • Mejora continua

📋 Métricas

  • • Lead Time
  • • Cycle Time
  • • Throughput
  • • WIP

  • • Tablero visual
  • • Límites WIP
  • • Políticas explícitas
  • • Cadencias de servicio

  • • Comunicación
  • • Simplicidad
  • • Feedback
  • • Coraje
  • • Respeto

🛠️ Prácticas

  • • Test-Driven Development
  • • Pair Programming
  • • Refactoring
  • • Simple Design

🔄 Procesos

  • • Planning Game
  • • Small Releases
  • • Continuous Integration
  • • Collective Code Ownership

🎯 Beneficios

  • • Alta calidad de código
  • • Respuesta rápida a cambios
  • • Reducción de defectos
  • • Satisfacción del cliente
